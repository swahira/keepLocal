# Contributing to localKeep

Thank you for your interest in localKeep! We welcome contributions to help make this the best local-first notes editor.

## How Can I Contribute?

### Reporting Bugs
- Use the [GitHub Issue Tracker](https://github.com/swahira/localKeep/issues) to report bugs.
- Provide a clear title and description.
- Include steps to reproduce the issue.

### Suggesting Enhancements
- Check if the feature has already been suggested.
- Describe the feature and why it would be useful.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the UI remains consistent with the existing aesthetic.
4. Update the documentation if necessary.

## Style Guide

### Code Style
- Use Vanilla JavaScript (ES6+).
- Follow a consistent naming convention (camelCase for variables/functions).
- Comment complex logic.

### Design Style
- Maintain the "premium" look (Inter font, JetBrains Mono for editor, consistent spacing).
- Respect the light/dark theme variables.

## Questions?
Feel free to open an issue for discussion.
